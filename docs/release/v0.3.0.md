# diag-lang v0.3.0 — Multi-line spans, secondary labels, notes

**The full frame.** v0.2.0 proved the hard part — a caret aligned exactly under a
single-line span. v0.3.0 builds the rest of the rustc-style frame on top of it: a
span that crosses lines, secondary labels pointing at related locations, and
trailing notes and help. The single-primary, single-line output is byte-for-byte
unchanged; everything here is additive.

## What is diag-lang?

The diagnostics layer of a compiler front-end. It collects diagnostics — a
severity, a message, and labelled spans — and renders them with source context, in
the style developers expect from a modern toolchain. It consumes positions from
`span-lang` and source text from `source-lang`, and owns presentation only.

## What's new in 0.3.0

### Secondary labels and notes

A `Diagnostic` now carries, beyond its primary label, any number of secondary
labels and trailing note/help lines, added with chainable builders:

```rust
use diag_lang::{Diagnostic, Label, Severity, Span};

let diag = Diagnostic::new(
    Severity::Error,
    "mismatched types",
    Label::new(Span::new(20, 27), "expected `i32`, found `&str`"),
)
.with_secondary(Label::new(Span::new(8, 11), "expected due to this"))
.with_note("string literals have type `&str`")
.with_help("convert with `.parse()`");
```

Secondary labels render with a `-` underline to set them apart from the primary
`^`. The `secondary`, `notes`, and `help` accessors read them back, for a consumer
that converts a diagnostic to another format rather than rendering it.

### Multi-line spans

A span that covers more than one line underlines every line it touches — the start
line from the span's start to the line's end, whole lines in the middle, and the
last line up to the span's end — with the message on the last:

```text
error: unbalanced delimiter
 --> m.rs:1:1
  |
1 | foo(
  | ^^^^
2 |   bar,
  | ^^^^^^
3 | )
  | ^ this group is never closed
  |
```

### Per-file frames, stacked underlines

Labels are grouped into one frame per source file — the file holding the primary
label first, the rest in the order they were added to the map. Within a frame,
each labelled line is shown once, with every label that touches it stacked
underneath, left to right:

```text
error: mismatched types
 --> main.rs:2:5
  |
1 | let x: i32 =
  |        --- expected due to this
2 |     "hi";
  |     ^^^^ expected `i32`, found `&str`
  |
```

The order is driven by where labels point — file, then line, then column — not by
the order they were added, so the same diagnostic always renders identically. A
secondary label in another file gets its own frame after the primary's.

### A defined fallback for every label

The v0.2.0 guarantee — a span outside the loaded sources renders a note rather than
panicking — now covers secondary labels too. An out-of-range label becomes a
`note:` line reporting its span, after the frames; the rest of the diagnostic
renders normally.

### Snapshot and property coverage

`tests/frames.rs` holds the new layouts byte for byte: multi-line spans,
same-file and cross-file secondary labels, two labels stacked on one line,
notes/help, the out-of-range note, and order-independence. `tests/properties.rs`
gains a property that rendering a diagnostic with arbitrary secondary labels across
several files stays total and deterministic.

## Breaking changes

**None.** Every addition is a new method or new rendering for input that did not
exist before; single-primary, single-line output is unchanged.

## Verification

Run on Windows x86_64 (Rust stable and 1.85 MSRV); the same commands run across the
CI matrix on Linux, macOS, and Windows:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo test
cargo test --all-features
cargo build --no-default-features
cargo +1.85 build --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo audit
cargo deny check
```

All green. Counts at this tag: 13 unit + 11 single-line snapshot + 7 frame snapshot
+ 4 property + 23 doctests.
