# diag-lang v1.0.0 — API freeze

**Stable.** v1.0.0 freezes the public API. Everything the 0.x series built — the
diagnostic model, the caret-aligned renderer, multi-line spans, secondary labels,
notes and help, and optional colour — is now under a Semantic Versioning promise:
no breaking change before `2.0`, additions only in minor releases, and the MSRV
(Rust 1.85) rising only in a minor. There are no functional API changes from
`0.4.0`; this release is the commitment, not new surface.

## What is diag-lang?

The diagnostics layer of a compiler front-end. It collects diagnostics — a
severity, a message, and labelled spans — and renders them with source context: the
offending lines with a caret underlining exactly where the problem is, in the style
developers expect from a modern toolchain. It consumes positions from `span-lang`
and source text from `source-lang`, and owns presentation only: a lexer, parser, or
type checker decides *what* is wrong, diag-lang decides how it looks.

```rust
use diag_lang::{Diagnostic, Label, Renderer, Severity, SourceMap, Span};

let mut map = SourceMap::new();
map.add("main.rs", "fn main() {\n    let x = foo();\n}\n").unwrap();

let diag = Diagnostic::new(
    Severity::Error,
    "cannot find value `foo` in this scope",
    Label::new(Span::new(24, 27), "not found in this scope"),
);

print!("{}", Renderer::new().render(&diag, &map));
```

```text
error: cannot find value `foo` in this scope
 --> main.rs:2:13
  |
2 |     let x = foo();
  |             ^^^ not found in this scope
  |
```

## The frozen surface

- **`Severity`** — `Error`, `Warning`, `Note`, `Help`, ordered most-to-least
  serious, with `as_str` and `Display`.
- **`Label`** — a global `Span` plus a message, via `new` / `unlabelled`.
- **`Diagnostic`** — a severity, a message, a required primary label, and
  chainable `with_secondary` / `with_note` / `with_help`, with accessors for each
  part.
- **`Renderer`** — `new`, `with_color` (the `color` feature), `render`, and
  `render_to` (the output seam, writing into any `core::fmt::Write`).
- **Re-exports** — `Span` and `SourceMap`, so a downstream need not name
  `span-lang` or `source-lang` to spell the types this API uses.

The full reference, including the SemVer promise and what is deliberately left out
(a `Render` trait, `serde`, configurable tab width — all addable later without a
breaking change), is in [`docs/API.md`](https://github.com/jamesgober/diag-lang/blob/main/docs/API.md).

## Guarantees

- **Alignment is exact.** The caret line is computed in display columns: one column
  per character over multi-byte UTF-8, tabs expanded to four-column stops. Proven by
  byte-exact snapshot tests and a property test against a naive column walk.
- **Rendering is total.** A span outside the loaded sources renders a defined note,
  never a panic. The same diagnostic always renders byte-identical output, and every
  render ends with a newline.
- **Colour only adds.** With styling off — the default — the plain output is
  complete and correctly aligned; stripping the ANSI escapes from a coloured render
  reproduces the plain bytes exactly.
- **`no_std`** with `alloc`; `#![forbid(unsafe_code)]`; no `unwrap`/`expect`/`todo!`
  in shipping code.

## Breaking changes

**None.** Identical API to `0.4.0`. From here, breaking changes require `2.0`.

## Verification

Run on Windows x86_64 (Rust stable and 1.85 MSRV); the same commands run across the
CI matrix on Linux, macOS, and Windows:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo clippy --no-default-features -- -D warnings
cargo test

cargo +1.85 build --all-features
```
