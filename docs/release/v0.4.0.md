# diag-lang v0.4.0 — Styling and the freeze candidate

**Colour, and the surface declared frozen.** v0.4.0 adds optional ANSI styling and
declares the public API complete: this is the surface that freezes at `1.0.0`. The
plain output is unchanged — colour only ever wraps it — so nothing built on v0.2.0
or v0.3.0 needs to change.

## What is diag-lang?

The diagnostics layer of a compiler front-end. It collects diagnostics — a
severity, a message, and labelled spans — and renders them with source context, in
the style developers expect from a modern toolchain. It consumes positions from
`span-lang` and source text from `source-lang`, and owns presentation only.

## What's new in 0.4.0

### Optional ANSI colour

The new `color` feature (on by default) adds `Renderer::with_color`, which styles
the output for a terminal: the severity word and primary underline in the level's
colour (red for an error, yellow for a warning, green for a note, cyan for help),
the frame gutter and secondary underlines in blue.

```rust
use diag_lang::Renderer;

// Plain by default — right for a file, a string, or a test.
let plain = Renderer::new();

// Coloured — right for a terminal.
let coloured = Renderer::new().with_color();
```

Colour is **additive**, and a test holds the whole crate to it: stripping the ANSI
escapes from a coloured render reproduces the plain render byte for byte. The
layout, the alignment, and the text never change — styling only wraps them — so a
sink that cannot interpret the codes loses nothing but the colour. A renderer that
never calls `with_color` emits no escapes at all.

### The output seam

`Renderer::render_to` — shipped in v0.2.0 — is the output seam: it writes into any
`core::fmt::Write`, so one renderer drives a terminal, a `String`, a buffer reused
across diagnostics, or a formatter, with no intermediate allocation. Combined with
`with_color`, the two named output targets — terminal and plain string — are both
covered without a separate type.

### The freeze candidate

`docs/API.md` now documents the complete public surface as the candidate for the
`1.0.0` freeze, with the SemVer promise that will hold from 1.0: no breaking change
before 2.0, additions in minors, MSRV rising only in a minor. It also records what
is deliberately left out and can be added later without breaking — a `Render` trait
(deferred while there is a single renderer; see `dev/NOTES.md`), `serde`
serialisation, and a configurable tab width.

## Breaking changes

**None.** `color` is additive and on by default; `Renderer::new()` still produces
the exact bytes v0.3.0 produced.

## Verification

Run on Windows x86_64 (Rust stable and 1.85 MSRV); the same commands run across the
CI matrix on Linux, macOS, and Windows:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo clippy --no-default-features -- -D warnings
cargo test
cargo test --all-features
cargo build --no-default-features
cargo +1.85 build --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo audit
cargo deny check
```

All green. Counts at this tag (default features): 13 unit + 11 single-line snapshot
+ 7 frame snapshot + 4 colour + 4 property + 24 doctests.
